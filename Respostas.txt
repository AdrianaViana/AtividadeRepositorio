

1 - o commit serve para controlar as versões do 
software ou seja é um conjunto de arquivos individuais que são ultilizados por todos que estão criado.
o programa sendo assim os programadores podem ver, rever, refazer alterações no código.

Exemplo: A programadora Camila fez um pluguim em sua área desktop e fez um sync(sincronizou para o repositório) para o repositório, e  depois veio a programadora Raquel e foi ver este pluguim e corrigiu alguns erros daí fez um commit e usou sync(sicronizou para o repositório).


2 - O botão sync serve para sincronizar os arquivos ou seja quando clico neste botão é enviado para o 
repositório os arquivos do programador que fez o commit.
